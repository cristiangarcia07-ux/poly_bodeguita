1. Análisis de Costes de Producción Críticos

Identifica de forma exhaustiva el plato que presenta el coste de fabricación más elevado dentro del catálogo de la bodega. Para este cálculo, considera exclusivamente la configuración de ración completa, calculada como el producto de la `cantidadracioncompletaenkg` por el `precio_x_euro` de cada ingrediente asociado. El informe resultante debe mostrar el nombre oficial del plato, el recuento total de ingredientes que lo componen y una columna única con la lista descriptiva de todos sus ingredientes concatenada por comas.

SELECT p.nombre, 
       COUNT(i.id) AS num_ingredientes,
       GROUP_CONCAT(i.descripcion SEPARATOR ', ') AS lista_ingredientes,
       ROUND(SUM(ip.cantidadracioncompletaenkg * i.precio_x_euro), 2) AS coste_fabricacion
FROM platos p
INNER JOIN ingrediente_plato ip ON p.id = ip.plato_id
INNER JOIN Ingredientes i ON ip.ingrediente_id = i.id
GROUP BY p.id, p.nombre
ORDER BY coste_fabricacion DESC
LIMIT 1;

2. Auditoría de Ingredientes sin Trazabilidad

Dentro del inventario de materias primas, existen elementos que parecen no haber tenido rotación en canales específicos. Identifica los nombres oficiales (`descripcion`) de aquellos ingredientes que NUNCA han formado parte de un pedido servido a domicilio (`es_a_domicilio = TRUE`) cuando dicho pedido ha sido gestionado por empleados cuyo nombre comience por la letra 'P' (ej. Paco, Pedro). Se requiere esta información para optimizar el almacenamiento y detectar posibles anomalías en la gestión de repartos.

SELECT i.descripcion
FROM Ingredientes i
WHERE i.id NOT IN (
    SELECT DISTINCT ip.ingrediente_id
    FROM ingrediente_plato ip
    INNER JOIN platos_pedidos pp ON ip.plato_id = pp.plato_id
    INNER JOIN pedidos p ON pp.pedido_id = p.id
    INNER JOIN empleados e ON p.empleado_id = e.id
    WHERE p.es_a_domicilio = TRUE 
    AND e.nombre LIKE 'P%'
);

3. Maximización del Margen de Beneficio

Determina qué cliente ha generado el mayor impacto económico positivo para la empresa sumando la rentabilidad de todas sus operaciones comerciales. El beneficio neto se debe calcular restando el coste total de fabricación (basado en la composición de ingredientes y el tipo de ración, media o completa) al precio de venta facturado. La consulta debe devolver el email institucional del cliente, su nombre completo y la métrica exacta del beneficio histórico generado tras la deducción de costes de producción.

WITH Ingresos AS (
    SELECT p.cliente_id, SUM(pp.precio_unitario * pp.cantidad) as total_ingreso
    FROM pedidos p
    INNER JOIN platos_pedidos pp ON p.id = pp.pedido_id
    GROUP BY p.cliente_id
),
Costes AS (
    SELECT p.cliente_id, SUM(ip.cantidadracioncompletaenkg * i.precio_x_euro * pp.cantidad) as total_coste
    FROM pedidos p
    INNER JOIN platos_pedidos pp ON p.id = pp.pedido_id
    INNER JOIN ingrediente_plato ip ON pp.plato_id = ip.plato_id
    INNER JOIN Ingredientes i ON ip.ingrediente_id = i.id
    WHERE pp.tipo_racion = 'completa'
    GROUP BY p.cliente_id
    UNION ALL
    SELECT p.cliente_id, SUM(ip.cantidadracionmediaenkg * i.precio_x_euro * pp.cantidad) as total_coste
    FROM pedidos p
    INNER JOIN platos_pedidos pp ON p.id = pp.pedido_id
    INNER JOIN ingrediente_plato ip ON pp.plato_id = ip.plato_id
    INNER JOIN Ingredientes i ON ip.ingrediente_id = i.id
    WHERE pp.tipo_racion = 'media'
    GROUP BY p.cliente_id
),
CostesAgrupados AS (
    SELECT cliente_id, SUM(total_coste) as coste_final
    FROM Costes
    GROUP BY cliente_id
)
SELECT c.email, c.nombre, COALESCE(ing.total_ingreso, 0) - COALESCE(ca.coste_final, 0) AS beneficio_global
FROM clientes c
INNER JOIN Ingresos ing ON c.id = ing.cliente_id
LEFT JOIN CostesAgrupados ca ON c.id = ca.cliente_id
ORDER BY beneficio_global DESC
LIMIT 1;

4. Concentración de Consumo de Aceite de Oliva

El departamento de compras ha notado un volumen de gasto singular en "Aceite de Oliva Virgen Extra". Necesitamos identificar al perfil de cliente que lidera el consumo de este producto. Localiza al cliente que ha acumulado la mayor cantidad total de platos servidos (`sum(cantidad)`) que incluyan de manera explícita dicho ingrediente en su receta. La respuesta debe incluir el nombre, los apellidos, el email de contacto y el total de raciones consumidas para su posterior perfilado comercial.

SELECT c.nombre, c.apellido1, c.email, SUM(pp.cantidad) AS total_raciones_aceite
FROM clientes c
INNER JOIN pedidos p ON c.id = p.cliente_id
INNER JOIN platos_pedidos pp ON p.id = pp.pedido_id
INNER JOIN ingrediente_plato ip ON pp.plato_id = ip.plato_id
INNER JOIN Ingredientes i ON ip.ingrediente_id = i.id
WHERE i.descripcion = 'Aceite de Oliva Virgen Extra'
GROUP BY c.id, c.nombre, c.apellido1, c.email
ORDER BY total_raciones_aceite DESC
LIMIT 1;

5. Desglose de Facturación por Canal y Empleado

Realiza un análisis comparativo de la eficiencia de ventas de la plantilla. Debes mostrar la facturación total gestionada por cada empleado (identificado por su `usuario`), desglosando la información en dos columnas diferenciadas: la recaudación proveniente exclusivamente de pedidos a domicilio (`Facturado_Domicilio`) y la recaudación generada en el local físico (`Facturado_Local`). Este desglose es fundamental para la toma de decisiones operativas sobre el personal de reparto.

WITH Domicilio AS (
    SELECT empleado_id, SUM(total) as domicilio
    FROM pedidos WHERE es_a_domicilio = TRUE GROUP BY empleado_id
), Local_Ped AS (
    SELECT empleado_id, SUM(total) as local_factura
    FROM pedidos WHERE es_a_domicilio = FALSE GROUP BY empleado_id
)
SELECT e.usuario, 
       COALESCE(d.domicilio, 0) AS facturado_domicilio, 
       COALESCE(l.local_factura, 0) AS facturado_local
FROM empleados e
LEFT JOIN Domicilio d ON e.id = d.empleado_id
LEFT JOIN Local_Ped l ON e.id = l.empleado_id;

6. Identificación de Pedidos Extremos

Para fines de auditoría financiera, se requiere obtener un extracto con los registros más dispares de la base de datos histórica. En una única consulta SQL, obtén exactamente dos registros: los detalles completos del pedido con el importe total más bajo y los del pedido con el importe total más alto. La salida debe incluir una etiqueta identificativa ("MAS BARATO" / "MAS CARO"), el ID del pedido, el importe total neto, el nombre del cliente y la dirección de entrega correspondiente.

(SELECT 'MAS BARATO' as tipo, p.id, p.total, c.nombre, d.Direccion
 FROM pedidos p
 INNER JOIN clientes c ON p.cliente_id = c.id
 LEFT JOIN Direcciones d ON p.direccion_id = d.id
 ORDER BY p.total ASC LIMIT 1)
UNION ALL
(SELECT 'MAS CARO' as tipo, p.id, p.total, c.nombre, d.Direccion
 FROM pedidos p
 INNER JOIN clientes c ON p.cliente_id = c.id
 LEFT JOIN Direcciones d ON p.direccion_id = d.id
 ORDER BY p.total DESC LIMIT 1);

7. Exclusividad Geográfica de Platos

Detecta patrones de consumo altamente localizados. Identifica aquellos platos que han sido solicitados ÚNICAMENTE por clientes cuya dirección de residencia esté vinculada al código postal '41004'. Si un plato ha sido incluido en al menos un pedido por un cliente residente en cualquier otro código postal, debe ser omitido del resultado. Este análisis permitirá evaluar la viabilidad de menús específicos por zona de reparto.

SELECT pl.nombre 
FROM platos pl
WHERE pl.id IN (
    SELECT DISTINCT pp.plato_id
    FROM platos_pedidos pp
    INNER JOIN pedidos p ON pp.pedido_id = p.id
    INNER JOIN cliente_direccion cd ON p.cliente_id = cd.cliente_id
    INNER JOIN Direcciones d ON cd.direccion_id = d.id
    WHERE d.codigo_postal = '41004'
) 
AND pl.id NOT IN (
    SELECT DISTINCT pp.plato_id
    FROM platos_pedidos pp
    INNER JOIN pedidos p ON pp.pedido_id = p.id
    INNER JOIN cliente_direccion cd ON p.cliente_id = cd.cliente_id
    INNER JOIN Direcciones d ON cd.direccion_id = d.id
    WHERE d.codigo_postal != '41004'
);

8. Análisis de Gasto en Dominios Específicos

Dentro de los perfiles de usuario corporativos, se desea auditar el volumen de gasto de los clientes con el dominio de correo electrónico '@ejemplo.com'. El requisito es calcular el gasto total acumulado de estos clientes, pero con una restricción técnica: solo se deben considerar aquellos pedidos que no incluyan ninguna ración de tipo 'completa'. Si un pedido contiene una ración completa, se debe ignorar por completo de este sumatorio de gasto ligero.

SELECT c.email, SUM(p.total) as GastoValido
FROM clientes c
INNER JOIN pedidos p ON c.id = p.cliente_id
WHERE c.email LIKE '%@ejemplo.com'
  AND p.id NOT IN (
      SELECT pedido_id FROM platos_pedidos WHERE tipo_racion = 'completa'
  )
GROUP BY c.email;

9. Densidad de Clientes y Volumen por Código Postal

Genera un informe demográfico-económico detallado por zona postal. La consulta debe devolver, para cada código postal registrado, el número total de clientes únicos asociados a dicha zona y el volumen total de créditos facturados (SUM de total pedidos) a dicho segmento. Los resultados deben presentarse ordenados de manera descendente según el volumen total facturado para identificar las áreas geográficas más rentables.

SELECT d.codigo_postal, 
       COUNT(DISTINCT c.id) AS recuento_clientes,
       COALESCE(SUM(p.total), 0) AS volumen_gasto_aportado
FROM Direcciones d
LEFT JOIN cliente_direccion cd ON d.id = cd.direccion_id
LEFT JOIN clientes c ON cd.cliente_id = c.id
LEFT JOIN pedidos p ON c.id = p.cliente_id
GROUP BY d.codigo_postal
ORDER BY volumen_gasto_aportado DESC;

10. Evaluación de Desempeño en Comandas Complejas

Se busca identificar a los empleados con mejor capacidad para gestionar pedidos de alta diversidad. Obtén el nombre completo del empleado, la cantidad total de pedidos que ha tramitado y el ingreso promedio por pedido. La condición es que solo deben entrar en el cálculo aquellos pedidos que contengan un mínimo de tres o más variedades de platos distintos (`DISTINCT plato_id`). Los pedidos estándar o de baja complejidad quedan fuera de esta métrica de rendimiento avanzado.

WITH PedidosMasDe3Platos AS (
    SELECT pedido_id
    FROM platos_pedidos
    GROUP BY pedido_id
    HAVING COUNT(DISTINCT plato_id) >= 3
)
SELECT e.nombre, 
       COUNT(p.id) AS numero_de_pedidos_grandes,
       AVG(p.total) AS media_dinero_por_pedido
FROM empleados e
INNER JOIN pedidos p ON e.id = p.empleado_id
INNER JOIN PedidosMasDe3Platos p3 ON p.id = p3.pedido_id
GROUP BY e.id, e.nombre;

11. Fidelización en Promoción Especial Jeremías

Se ha activado una campaña de fidelización para clientes recurrentes de nombre 'Jeremías'. La promoción consiste en otorgar una bonificación a aquellos clientes cuyo nombre comience por 'jeremias%' y que hayan solicitado pedidos a domicilio a una misma dirección en 20 o más ocasiones. Identifica cuántos pedidos ha realizado cada cliente que cumpla el patrón de nombre a cada una de sus direcciones, indicando claramente cuáles de estas combinaciones superan el umbral de la promoción.

SELECT c.nombre, d.Direccion, COUNT(p.id) AS recuento_pedidos
FROM clientes c
INNER JOIN pedidos p ON c.id = p.cliente_id
INNER JOIN Direcciones d ON p.direccion_id = d.id
WHERE c.nombre LIKE 'jeremias%' AND p.es_a_domicilio = TRUE
GROUP BY c.id, c.nombre, d.id, d.Direccion
HAVING COUNT(p.id) >= 20
ORDER BY recuento_pedidos DESC;

12. Gestión de Comandas Automatizada (Apertura)

Inicia el proceso de automatización de ticketing mediante el uso de procedimientos almacenados. Utiliza el procedimiento `AbrirPedido` para crear una nueva comanda registrada a nombre del cliente con email 'jeremiaselcocas@gmail.com'. El mozo responsable asignado debe ser 'cruiz' y la entrega se realizará a domicilio en la dirección estratégica de Málaga situada en 'Calle Larios 12'. Asegúrate de capturar los identificadores correspondientes mediante sentencias SET antes de ejecutar la llamada al procedimiento.

SET @cliente = (SELECT id FROM clientes WHERE email = 'jeremiaselcocas@gmail.com');
SET @empleado = (SELECT id FROM empleados WHERE usuario = 'cruiz');
SET @direccion = (SELECT id FROM Direcciones WHERE Direccion = 'Calle Larios 12');
CALL AbrirPedido(@cliente, @empleado, @direccion, TRUE);

13. Modificación de Comanda: Lógica de Cortesía

Para mejorar la satisfacción del cliente, se debe añadir una línea de cortesía al pedido generado en el paso anterior. Utiliza el procedimiento especializado `AñadirPlatoAPedido` para integrar 5 raciones de tipo 'completa' del plato 'Secreto Ibérico'. Es imperativo configurar el parámetro de cortesía (`gratis = true`) para asegurar que el sistema no incremente el subtotal facturable, manteniendo la trazabilidad del consumo sin cargo adicional para el cliente.

SET @pedido = (SELECT MAX(id) FROM pedidos);
SET @plato = (SELECT id FROM platos WHERE nombre = 'Secreto Ibérico');
CALL AñadirPlatoAPedido(@pedido, TRUE, @plato, 'completa', 5);

14. Finalización y Recálculo Impositivo del Ticket

Concluye la edición del pedido anterior añadiendo productos estándar con cargo. Emplea de nuevo el procedimiento `AñadirPlatoAPedido` para sumar 2 raciones completas de 'Chuletas de Cordero', esta vez con el parámetro de cortesía desactivado (`gratis = false`). Verifica que el sistema recalcula con precisión el subtotal actualizado y aplica de manera automática el devengo del 21% de IVA sobre el total final de la operación, cerrando así la cadena de gestión impositiva.

SET @pedido = (SELECT MAX(id) FROM pedidos);
SET @plato = (SELECT id FROM platos WHERE nombre = 'Chuletas de Cordero');
CALL AñadirPlatoAPedido(@pedido, FALSE, @plato, 'completa', 2);
